# Blinker

軽量・高速起動の Windows 用画像ビューア。C++20 / Win32 API / Direct2D / WIC 製、外部依存ゼロの単一 exe。

## 特徴

- **軽快**: 数百 KB の単一実行ファイル。起動即表示、フォルダ内の画像を先読みして瞬時に遷移
- **キーボード中心**: すべての操作にショートカットキー。`blinker.ini` で自由に変更可能
- **対応形式**: JPEG / PNG / BMP / GIF / TIFF / ICO / JXR。Windows 11 なら HEIC / WebP / AVIF も(OS コーデック経由)
- **拡張しやすい設計**: プラットフォーム非依存の core 層と Win32 実装層を分離(詳細は [docs/architecture.md](docs/architecture.md))

## ビルド

必要環境: Visual Studio 2022(「C++ によるデスクトップ開発」ワークロード)

```powershell
# "Developer PowerShell for VS 2022" で実行
cmake --preset release
cmake --build --preset release
# 生成物: build/release/blinker.exe

# テスト
ctest --preset release
```

## 使い方

```
blinker.exe <画像ファイル | フォルダ>
```

画像ファイルへのドラッグ&ドロップ、「プログラムから開く」にも対応。

### デフォルトキーバインド

| 操作 | キー |
|---|---|
| 次 / 前の画像 | `→` `↓` `Space` `PageDown` / `←` `↑` `Backspace` `PageUp` |
| 最初 / 最後の画像 | `Home` / `End` |
| ズームイン / アウト | `+` / `-`、マウスホイール(カーソル位置基準) |
| ウィンドウにフィット / 等倍 | `0` / `1` |
| スクロール(パン) | `Ctrl+矢印`、マウス左ドラッグ |
| 回転(表示のみ) | `R`(右90°)/ `Shift+R`(左90°) |
| フルスクリーン | `F11` `Enter` |
| ファイルを開く | `Ctrl+O` |
| 画像をクリップボードへコピー | `Ctrl+C` |
| 画像のパスをコピー | `Shift+Ctrl+C` |
| クリップボードの画像を貼り付け | `Ctrl+V` |
| 名前を付けて保存 | `Ctrl+S` |
| 編集を元に戻す | `Ctrl+Z` |
| 選択中の図形・テキストを削除 | `Delete` |
| サイドバー(ファイル名一覧)表示切替 | `Ctrl+B` |
| ステータスバー表示切替 | `B` |
| 終了 | `Q` `Ctrl+W`、`Esc`(フルスクリーン中は解除) |

### 画像編集

マウスの**右ドラッグ**で画像上の領域を選択すると編集メニューが開く。

- **トリミング**: 選択領域で画像を切り出す(図形・テキストは位置を保ったまま残る)
- **図形**: 矩形 / 楕円 / 矢印 / 直線を選択領域に描く
- **テキスト**: 入力ダイアログで文字列を書き込む(`Enter` で改行できる複数行対応。
  選択領域の幅で自動折り返し)

メニュー下部の設定はその場で切り替えられ、選んだ後もメニューが再表示されるので
続けて図形やテキストを描ける(設定は新規作成時の既定値になる):

- **線の太さ / 文字サイズ**: プリセットから選択(画面上の見た目基準。ズーム率に応じて
  画像へ換算される)
- **色の変更...**: Windows 標準の色ダイアログで自由に指定

追加した図形・テキストはオブジェクトとして残り、あとから編集できる:

- **左クリック**で選択(破線の枠とハンドルが出る)。**左ドラッグ**で移動
- 枠の上に出る**回転ハンドル**をドラッグすると自由な角度に回転(`Shift` で15°刻み)
- **右クリック**でオブジェクトメニューが開き、回転角度(プリセット)・線の太さ・
  文字サイズ・色の変更・削除ができる
- テキストは**ダブルクリック**(またはオブジェクトメニュー)で内容を再編集できる
- `Delete` で選択中のオブジェクトを削除。`Esc` で選択解除

編集は `Ctrl+Z` で1段階ずつ取り消せる(履歴は直近10回)。図形・テキストは画像へは
焼き込まれず、`Ctrl+S` の保存や `Ctrl+C` のコピー時に合成される。元ファイルが自動で
書き換わることはなく、別の画像へ移動すると未保存の編集は破棄される。
線の太さ・文字サイズ・色の初期値は `blinker.ini` の `[edit]` セクションで変更できる。

### 設定

`blinker.exe` と同じフォルダに `blinker.ini` を置くと、キーバインドや背景色、
タイトルバーのテーマ(OS設定追従 / ダーク / ライト)を変更できる。
書式は [docs/blinker.ini.example](docs/blinker.ini.example) を参照。
